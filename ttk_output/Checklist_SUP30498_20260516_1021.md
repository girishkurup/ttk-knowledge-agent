# Compression Scale-Up Checklist -- Derived from Batch SUP30498
**Product:** Enalapril Tablets (OSD) | **Applicable Stage:** Pilot-to-Commercial Tech Transfer
**Reference KA:** KA-260516-SUP30498 | **Version:** 1.0 (Interim -- pending pilot study close-out)

---

## Pre-Compression (Materials)

- [ ] Confirm granule LOD result for the specific sub-lot entering compression; do not proceed if LOD < 1.6% without escalation
- [ ] Verify that the granule LOD target for this batch is set at 1.8% (not the legacy pilot target of 1.2%)
- [ ] Document actual LOD value on BMR and compare against both the specification limit and the recommended commercial target range
- [ ] Confirm granule LOD was measured within 4 hours of compression start; repeat measurement if granules have been held in open IBC or under non-controlled RH conditions
- [ ] Review granulation batch record for any deviations in fluid bed or high-shear endpoint that could have produced a bimodal LOD distribution within the batch
- [ ] Confirm no changes to excipient grade or supplier versus the pilot formulation (particularly for binders and lubricants affecting plasticity)
- [ ] Verify lubricant blending time and speed were within the validated range -- over-lubrication reduces compactability and compounds capping risk

---

## Pre-Compression (Equipment)

- [ ] Document commercial press model, turret diameter, number of stations, and punch head flat dimension on the compression setup sheet
- [ ] Retrieve pilot press punch head flat dimension from tech transfer package and calculate dwell time ratio (commercial vs. pilot) at the planned operating speed
- [ ] Confirm dwell time at planned commercial RPM is within ± 20% of pilot dwell time; if delta exceeds 20%, escalate to process owner before proceeding
- [ ] Verify punch tooling is identical in head flat geometry to the pilot tooling specification, or document and risk-assess any differences
- [ ] Inspect all punches and dies for wear, chips, or dimensional non-conformance before fitting; record punch set ID and inspection outcome
- [ ] Confirm feed frame paddle speed and fill cam depth are set to parameters validated for this product at commercial scale (not copied directly from pilot without adjustment)
- [ ] Calibrate compression force transducers and verify upper/lower punch force balance before run start
- [ ] Verify pre-compression force is set within range to aid air evacuation from the die -- critical for capping-prone formulations
- [ ] Confirm forced feeder is operational and set to maintain consistent die fill without over-shearing the granule

---

## Scale-Up Specific Considerations

- [ ] Calculate and document expected dwell time at minimum, target, and maximum turret speed for the commercial press; file in batch record
- [ ] Set initial turret speed at or below 58 RPM for first commercial batch of this product; do not increase beyond 58 RPM without in-process capping assessment at each speed increment
- [ ] Confirm the compression suite RH specification is ≤ 45% RH (or per product-specific limit); obtain environmental release sign-off before granule transfer
- [ ] Confirm RH sensor at die table level is calibrated and functional; this is a mandatory monitoring point for this product
- [ ] Verify that site environmental control system is stable (no ongoing HVAC maintenance or seasonal changeover) before scheduling this compression run
- [ ] Confirm that the pilot-to-commercial tech transfer package includes a formal dwell time comparison; if this section is absent, raise a gap and obtain process owner sign-off before proceeding
- [ ] Ensure that the compression BMR references the current interim LOD target of 1.8% and not an older version referencing 1.2%

---

## During-Run Monitoring Triggers

- [ ] Perform a dedicated visual capping inspection at T = 15 min, T = 30 min, and T = 45 min from compression start -- do not rely solely on automated rejection data in the first hour
- [ ] Sample 20 tablets at T = 45 min specifically for visual defect assessment (capping, lamination, chipping); document result on in-process log
- [ ] Monitor and record die table RH at minimum every 30 minutes throughout the run; log against alert limit
- [ ] Monitor compression force mean and variability (RSD) at each in-process check; flag if force RSD exceeds the alert threshold defined in the BMR
- [ ] Record tablet weight and hardness at every scheduled in-process check; flag any trend toward increasing weight variability (early indicator of feed frame issues contributing to force variability)
- [ ] If ambient RH rises above the alert limit at any point during the run, immediately perform an unscheduled capping inspection and notify the process owner
- [ ] If capping frequency increases between any two consecutive inspection points, stop and assess before continuing -- do not allow the run to self-correct without documented evaluation

---

## Escalation / Red Flag Criteria

- [ ] **STOP AND ESCALATE** if capping rate exceeds 0.5% of tablets inspected at any single in-process check point
- [ ] **STOP AND ESCALATE** if capping is first observed after T = 30 min (mid-run onset is the signature failure mode of this batch -- treat as process drift, not a startup issue)
- [ ] **STOP AND ESCALATE** if die table RH exceeds the defined action limit at any point during the run
- [ ] **STOP AND ESCALATE** if compression force RSD exceeds the BMR action limit -- force variability is a confirmed co-factor in capping for this product
- [ ] **REDUCE SPEED TO 50 RPM** (and document as a deviation with immediate notification to process owner) if initial capping inspection at T = 45 min shows any capping; do not first try increasing speed or force
- [ ] **DO NOT** attempt to resolve capping by increasing main compression force alone without also evaluating granule LOD and RH -- force increase without plasticity improvement will not resolve the root cause and may cause punch damage
- [ ] If speed reduction and LOD assessment do not resolve capping within two consecutive in-process checks, halt compression and initiate formal deviation before continuing

---

## Documentation Requirements

- [ ] Record actual granule LOD (specific sub-lot measurement, date, and instrument ID) in the BMR before compression start sign-off
- [ ] Record initial turret speed set point and any speed changes made during the run, with timestamps and rationale
- [ ] Record die table RH readings at each monitoring interval in the in-process log (not only flag exceptions -- log all readings)
- [ ] If any mid-run parameter adjustment is made (speed, force, feed frame), raise an in-process deviation notification immediately; do not defer to end-of-batch documentation
- [ ] Document punch set ID, head flat dimension, and pre-run inspection outcome in the equipment use log
- [ ] Complete a post-compression capping summary: total tablets rejected, time distribution of capping events, and parameter values at time of each event
- [ ] Forward completed in-process monitoring data to process owner within 24 hours of batch completion for trend review against SUP30498 baseline
- [ ] If this is the first or second commercial batch of this product, schedule a structured knowledge capture debrief with the lead compression operator within 48 hours of batch completion
