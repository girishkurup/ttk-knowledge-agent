# Compression Scale-Up Checklist -- Derived from Batch SUP30498
**Product:** Enalapril Tablets (OSD)
**Applicable Equipment:** Korsch XL800 (or equivalent commercial-scale rotary press)
**Reference KA:** KA-260515-SUP30498
**Version:** 1.0 | **Date:** 2026-05-15

---

## Pre-Compression (Materials)

- [ ] Confirm blend LOD result is documented and reviewed **before** compression is initiated — do not rely solely on release-to-compress status
- [ ] Verify blend LOD is **≥ 1.8%** for Enalapril Tablets; if LOD is 1.6–1.79%, place blend on hold and escalate to process engineering for disposition before proceeding
- [ ] Confirm blend LOD test was performed on the commercial-scale blended lot — do not extrapolate from granulation-step LOD alone
- [ ] Review blend particle size distribution and bulk/tapped density against pilot reference values; flag deviations >10% for assessment
- [ ] Confirm API lot is the same grade and supplier as validated pilot material; if changed, verify elastic recovery characterisation data is on file
- [ ] Confirm lubricant blend time and concentration match the validated commercial procedure (over-lubrication will reduce tablet tensile strength and worsen capping tendency)

---

## Pre-Compression (Equipment)

- [ ] Confirm press model and turret configuration match the compression procedure (XL800, correct tooling set)
- [ ] Record lower punch penetration depth set-point in equipment setup log — **commercial set-point is pilot default + 0.2 mm deeper** (per SUP30498 finding); verify this value is loaded, not the pilot default
- [ ] Verify all punch and die tooling has been inspected, dimensionally verified, and logged within the current maintenance cycle
- [ ] Confirm punch positions 12 and 13 (upper row) have been individually inspected for chips, wear, or surface irregularities prior to campaign start
- [ ] Verify feed frame paddles are free of wear or damage that could cause uneven die fill
- [ ] Confirm metal detector calibration is current and challenge-tested with appropriate test pieces before run start
- [ ] Verify compression force transducer calibration is current for both pre-compression and main compression stations
- [ ] Confirm turret speed set-point is entered as **≤ 28 RPM** for this formulation; document any deviation from this ceiling with process engineering approval
- [ ] Calculate and document effective dwell time at the planned turret speed — record in batch record alongside compression force values

---

## Scale-Up Specific Considerations

- [ ] Confirm dwell time at commercial turret speed has been formally calculated (using punch head flat length, turret diameter, and RPM) and compared to pilot dwell time — document the delta
- [ ] Confirm pre-compression force has been independently optimised for the XL800 — **do not simply carry over pilot value**; minimum starting point per SUP30498 resolution is **4.5 kN**
- [ ] Verify that main compression force (12 kN target), pre-compression force (4.5 kN), and turret speed (≤ 28 RPM) are all set to commercial-validated values before run start — confirm no single parameter has been silently reverted to pilot defaults
- [ ] Confirm feed frame speed set-point is documented and set to the value established during SUP30498 resolution (retrieve from press data log KA-260515-SUP30498 appendix)
- [ ] Conduct a minimum 500-tablet warm-up / equilibration run before beginning formal in-process sampling; discard warm-up tablets
- [ ] Collect and test the first formal in-process sample (weight, hardness, thickness, friability, disintegration) at tablet 1,000 — do not wait for standard interval if early-run rejections are observed

---

## During-Run Monitoring Triggers

- [ ] **Weight CV > 1.2%** at any in-process check → immediately inspect feed frame speed and lower punch penetration depth before continuing
- [ ] **Rejection rate > 1.0%** (metal detector or manual visual) at any point → pause run, inspect punch positions 12 & 13 first, then evaluate turret speed and pre-compression force
- [ ] **Lamination or capping observed on any ejected tablet** → stop run; do not adjust parameters and continue without engineering review
- [ ] **Hardness trending downward across consecutive in-process checks** (>0.5 kP drop) without a compression force change → flag for evaluation of moisture-related compaction loss
- [ ] **Punch ejection force increasing over time** → inspect lower punch tips and die bore for product pick-up or galling; do not compress through increasing ejection force
- [ ] Monitor and log turret speed, pre-compression force, and main compression force every 30 minutes; confirm no auto-compensation drift has altered set-points
- [ ] Conduct structured operator debrief observation at the 1-hour mark — ask operators explicitly whether any informal adjustments have been made and log responses

---

## Escalation / Red Flag Criteria

- [ ] **Blend LOD < 1.8%** — Do not compress; escalate to process engineering and QA
- [ ] **Rejection rate ≥ 2%** sustained over 500 tablets after one parameter adjustment attempt — Stop compression; initiate formal deviation; do not continue trialling adjustments without engineering sign-off
- [ ] **Any full cap-off (crown separation) event** — Stop compression immediately; 100% visual inspection of tablets produced in the preceding 1,000-tablet window; initiate deviation
- [ ] **Weight CV > 2.0%** at any in-process check — Stop compression; do not release any tablets from the affected segment without individual weight verification
- [ ] **Operator reports any undocumented equipment adjustment** — Pause run; document the adjustment formally before continuing; assess impact on batch integrity
- [ ] **Punch position clustering** of defects at any two adjacent positions — Escalate to engineering; do not assume tooling is the cause without inspection; evaluate turret dynamics and feed frame uniformity
- [ ] **LOD result received post-compression start showing < 1.8%** — Quarantine all tablets compressed from that blend lot pending process engineering review

---

## Documentation Requirements

- [ ] Record calculated dwell time (in milliseconds) in the batch record alongside turret speed
- [ ] Record lower punch penetration depth as a named parameter in the equipment setup section of the batch record — value must be explicitly stated, not referenced as 'per pilot'
- [ ] Record pre-compression force set-point and any in-run adjustments with timestamp and authorising engineer name
- [ ] Record feed frame speed set-point and any in-run adjustments with timestamp
- [ ] Log all in-process weight, hardness, and rejection data at each sampling point with pass/fail determination
- [ ] Document any operator-initiated adjustment (including those within equipment operating range) in a dedicated 'operator observations' field in the batch record
- [ ] Complete structured operator debrief form at end of run — minimum fields: adjustments made, observations not captured in automated data, any anomalies noted
- [ ] Attach punch inspection records for all tooling positions, with individual records for positions 12 & 13 (upper row)
- [ ] File this checklist as a completed attachment to the batch record upon campaign close
